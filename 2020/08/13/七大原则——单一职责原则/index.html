七大原则——单一职责原则

2020-08-13- 2020-08-24
13k- 26 分钟-
设计模式 设计模式七大原则

一、什么是单一职责原则

  对于类来说，即一个类应该只负责其对应的一个职责。怎样理解？先来看一个问题

未使用单一职责原则可能出现的问题

  例如有一个类T负责两个不同的职责：职责P1和职责P2。当因为职责P1的需求发生改变而需要修改类T的时候，有可能会导致原本运行正常的职责P2功能发生故障。

使用单一职责原则后

  遵循单一职责原则，分别建立两个类T1、T2，使T1完成职责P1功能，T2完成职责P2功能。这样，当修改类T1的时候，不会使职责P2发生故障风险。同理，当修改T2的时候，也不会使职责P1发生故障风险。

  注意：这个一项职责不是说一个类中只能有一个方法，而是其类所对应的一些行为，比如订单类会有订单的行为，而客户类中只有客户的一些行为

二、使用单一职责的案例

1、初始代码

package com.blog.principle.singleResponsibility;

public class SingleResponsibility01 {
    public static void main(String[] args) {
        Teacher.teach("数学老师");
        Teacher.teach("语文老师");
        Teacher.teach("体育老师");
    }
}

class Teacher{
    public static void teach(String teacherName) {
        System.out.println(teacherName + "在教数学");
    }
}

运行结果：

数学老师在教数学
语文老师在教数学
体育老师在教数学

2、改动后:

方案一：

package com.blog.principle.singleResponsibility;

public class SingleResponsibility02 {
    public static void main(String[] args) {
        TeacherMath.teach("数学老师");
        TeacherChinese.teach("语文老师");
        TeacherEnglish.teach("英语老师");
    }
}

class TeacherMath{
    public static void teach(String teacherName) {
        System.out.println(teacherName + "在教数学");
    }
}

class TeacherChinese{
    public static void teach(String teacherName) {
        System.out.println(teacherName + "在教语文");
    }
}

class TeacherEnglish{
    public static void teach(String teacherName) {
        System.out.println(teacherName + "在教英语");
    }
}

运行结果：

数学老师在教数学
语文老师在教语文
英语老师在教英语

总结

  虽然遵循了单一职责原则但是改动地方太多，工程量大，成本高

方案二：

package com.blog.principle.singleResponsibility;

public class SingleResponsibility03 {
    public static void main(String[] args) {
        Teach.ChineseTeacher("语文老师");
        Teach.MathTeacher("数学老师");
        Teach.EnglishTeacher("英语老师");
    }
}

class Teach{
    public static void MathTeacher(String teacherName) {
        System.out.println(teacherName + "在教数学");
    }

    public static void ChineseTeacher(String teacherName) {
        System.out.println(teacherName + "在教语文");
    }

    public static void EnglishTeacher(String teacherName) {
        System.out.println(teacherName + "在教英语");
    }
}

运行结果：

数学老师在教数学
语文老师在教语文
英语老师在教英语

总结

  这种改动没有对原来的类做很大修改，只是增加了一些方法
虽然没有在类这个级别上遵循单一职责原则，但是做到了方法级别的单一职责原则

三、使用单一职责原则的好处

  1. 降低类的复杂度，一个类只负责对应的职责
  2. 提高类的可读性
  3. 降低需求改动所带来的高成本维护

四、什么时候使用单一职责原则

通常情况下：我们都需要遵守单一职责原则

只有逻辑足够简单，才可以在代码级别违反单一原则。只有类中的方法数量足够少，才可以在方法级别保持单一原则

七大原则——依赖倒转原则

2020-08-13- 2020-08-24
31k- 1:03-
设计模式 设计模式七大原则

一、基本介绍

依赖倒转(Dependence Inversion Principle, DIP)原则是指：

  1. 高层模块不应该依赖低层模块，两者都应该依赖其抽象

     模块间的依赖通过抽象发生，实现类之间不发生直接的依赖关系，其依赖关系是通过接口或抽象类产生的

  2. 抽象不应该依赖细节、细节应该依赖抽象

     接口或抽象类不依赖于实现类

  3. 依赖倒转原则的中心思想是面向接口编程

     实现类依赖于接口或抽象类

依赖倒转原则带来的好处

  相对于细节的多变性，抽象的东西更加的稳定，以抽象为基础搭建的架构比以细节为基础搭建的架构要稳定的多，这里的抽象大多指的是接口和抽象类，而细节就是其具体的实现类。采用依赖倒置原则可以减少类间的耦合性，提高系统的稳定性，降低并行开发引起的风险，提高代码的可读性和可维护性

二、案例实现

案例一：现有需求如下：定义一个pos机要求其能刷农业银行的银行卡

package com.blog.principle.dependenceInversion;

public class DependenceInversion01 {
    public static void main(String[] args) {
        POS pos = new POS();
        pos.PayByCard(new ABCBank());
    }
}

class ABCBank{
    public void getMoney() {
        System.out.println("农业银行取款");
    }
}

class POS{
    public void PayByCard(ABCBank ABCBank) {
        ABCBank.getMoney();
    }
}

运行结果：

    农业银行取款

案例二：现有增加需求，要求能刷多个银行的银行卡如工商银行，建设银行等…

  方案一：

package com.blog.principle.dependenceInversion;

public class DependenceInversion01 {
    public static void main(String[] args) {
        POS pos = new POS();
        pos.PayByCard(new ICBCBank());
    }
}

class ABCBank{
    public void getMoney() {
        System.out.println("农业银行取款");
    }
}

class ICBCBank{
    public void getMoney() {
        System.out.println("工商银行取款");
    }
}

class CCBBank{
    public void getMoney() {
        System.out.println("建设银行取款");
    }
}

class POS{
    public void PayByCard01(ABCBank ABCBank) {
        ABCBank.getMoney();
    }

    public void PayByCard02(ICBCBank ICBCBank) {
        ICBCBank.getMoney();
    }

    public void PayByCard03(CCBBank CCBBank) {
        CCBBank.getMoney();
    }
}

运行结果

    工商银行取款

虽然简单的实现了功能但是要修改的地方太多，每次需要增加银行都需要新写一个类，同时还需要增加其对应的方法。扩展性不高，且代码重复性很高

  方案二：

package com.blog.principle.dependenceInversion;

public class DependenceInversion01 {
    public static void main(String[] args) {
        POS pos = new POS();
        pos.PayByCard(new ICBCBank());
    }
}

interface Money{
    void getMoney();
}

class ABCBank implements Money{
    @Override
    public void getMoney() {
        System.out.println("农业银行取款");
    }
}

class ICBCBank implements Money{
    @Override
    public void getMoney() {
        System.out.println("工商银行取款");
    }
}

class CCBBank implements Money{
    @Override
    public void getMoney() {
        System.out.println("建设银行取款");
    }
}

class POS{
    public void PayByCard(Money Money) {
        Money.getMoney();
    }
}

运行结果

    工商银行取款

使用这种方式以后，如果还需要增加其他的银行的话，只需要实现对应的Money接口即可

三、依赖倒转的三种传递方式

1、接口传递

package com.blog.principle.dependenceInversion;

public class DependenceInversion02 {
    public static void main(String[] args) {
        //使用接口传递
        OpenAndClose openAndClose = new OpenAndClose();
        //将需要打电话的手机传入
        openAndClose.open(new MiPhone());
    }
}

interface IOpenAndClose{
    void open(IPhone phone);
}

interface IPhone{
    void call();
}

class MiPhone implements IPhone{
    @Override
    public void call() {
        System.out.println("使用小米手机打电话");
    }
}

class OpenAndClose implements IOpenAndClose{
    @Override
    public void open(IPhone phone) {
        phone.call();
    }
}

运行结果

    使用小米手机打电话

2、构造方法传递

package com.blog.principle.dependenceInversion;

public class DependenceInversion02 {
    public static void main(String[] args) {
        //通过构造方法传递    将需要打电话的手机传递到有参构造方法中
        OpenAndClose openAndClose = new OpenAndClose(new HUAWEIPhone());
        openAndClose.open();
    }
}

interface IOpenAndClose{
    void open();
}

interface IPhone{
    void call();
}

class HUAWEIPhone implements IPhone{
    @Override
    public void call() {
        System.out.println("使用华为手机打电话");
    }
}

class OpenAndClose implements IOpenAndClose{
    private IPhone phone;

    public OpenAndClose() {
    }

    public OpenAndClose(IPhone phone) {
        this.phone = phone;
    }

    @Override
    public void open() {
        this.phone.call();
    }
}

运行结果

    使用华为手机打电话

3、setter方法传递

package com.blog.principle.dependenceInversion;

public class DependenceInversion02 {
    public static void main(String[] args) {
        //使用set方法传递
        OpenAndClose openAndClose = new OpenAndClose();
        //将打电话的手机传入
        openAndClose.setPhone(new OPPOPhone());
        //打开手机打电话
        openAndClose.open();
    }
}

interface IOpenAndClose{
    void open();
}

interface IPhone{
    void call();
}

class OPPOPhone implements IPhone{
    @Override
    public void call() {
        System.out.println("使用OPPO手机打电话");
    }
}

class OpenAndClose implements IOpenAndClose{
    private IPhone phone;

    public void setPhone(IPhone phone) {
        this.phone = phone;
    }

    @Override
    public void open() {
        this.phone.call();
    }
}

运行结果

    使用OPPO手机打电话

四、注意事项

  1. 低模块尽量有抽象类或者接口，也可以两种都有
  2. 变量的声明类型尽量用抽象类和接口，可以在变量引用和实际对象中间产生一个缓冲层，便于优化和扩展
  3. 继承的时候遵循里氏替换原则
